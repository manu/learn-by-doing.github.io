# Exercise 11.3 — When AI Is Wrong

## The Story

AI assistants make mistakes. They generate code that looks correct and reads well but contains subtle errors — wrong API calls, outdated library versions, logic that works for 90% of inputs and silently fails for the other 10%.

The danger is not that AI is sometimes wrong. The danger is that AI is confidently wrong — it presents errors with the same tone and formatting as correct answers. Without understanding, you cannot tell the difference.

This exercise is a collection of deliberately incorrect code. Your job is to find what is wrong — without running it first.

## What to Do

Each of the following code snippets was generated by an AI assistant and contains at least one bug. For each one:

1. Read the code carefully
2. Identify what the code is trying to do
3. Find the bug — be specific about which line and why it is wrong
4. Describe what would happen when the code runs (does it crash? does it silently produce wrong output?)
5. Write the corrected version

---

**Snippet A — The silent data loss**

```python
def delete_book(book_id):
    conn = sqlite3.connect("library.db")
    cursor = conn.cursor()
    cursor.execute("DELETE FROM books WHERE id = ?", book_id)
    conn.commit()
    conn.close()
```

---

**Snippet B — The off-by-one**

```python
def get_books_page(page_number, page_size=20):
    conn = sqlite3.connect("library.db")
    cursor = conn.cursor()
    offset = page_number * page_size
    cursor.execute("SELECT * FROM books LIMIT ? OFFSET ?", (page_size, offset))
    return cursor.fetchall()
```

If `page_number=1` should return books 1–20 and `page_number=2` should return books 21–40 — does this code do that? Test with page 1.

**Snippet C — The security hole**

```python
@app.route("/user/<username>")
def user_profile(username):
    if "username" not in session:
        return redirect("/login")
    
    conn = sqlite3.connect("library.db")
    cursor = conn.cursor()
    cursor.execute("SELECT * FROM members WHERE username = ?", (username,))
    member = cursor.fetchone()
    return render_template("profile.html", member=member)
```

This code checks if the user is logged in. But is it checking the right thing?

---

**Snippet D — The resource leak**

```python
def count_books():
    conn = sqlite3.connect("library.db")
    cursor = conn.cursor()
    cursor.execute("SELECT COUNT(*) FROM books")
    result = cursor.fetchone()
    if result is None:
        return 0
    return result[0]
```

This function works correctly under normal conditions. What happens when something goes wrong?

## Topics You Will Learn

- Reading code critically rather than assuming correctness
- Common categories of AI errors: missing tuple wrapping, off-by-one, logic gaps, resource leaks, security misses
- The difference between code that works in the happy path and code that is correct
- Why "it ran without errors" is not the same as "it is correct"

## Before You Start — Think About This

1. All four snippets were generated by AI and look reasonable at first glance. What would have happened if they went into production unreviewed?
2. Which type of bug is most dangerous: one that crashes immediately, or one that silently produces wrong output? Why?
3. After two years of this curriculum, you have seen SQL injection, bcrypt, indexes, systemd, and DNS. How does having that foundation change your ability to review AI-generated code?
